### 2. 戦闘

```
     T
Hero    Enemy
♥99    99♥
🛡        🛡
●        ●
棒        棒

```

`T`はターン数。`Hero`はユーザ、`Enemy`はCPU。♥はライフで、これが0になったほうの負け。それ以降の`🛡`は盾、`●`は爆弾、棒、薬という特殊なアイテム的何か。

以下は手札。手札は毎ターン、ランダムで3枚出現する。その中から1枚を選ぶ。常に`Hero`が先手で、`Enemy`が後手。

```
叩く	敵♥-1。前回攻撃済みなら敵♥-1。
殴る	敵♥-2。次回"持つ","拾う"不可
蹴る	敵♥-2。次回"躱す","誘う"不可
弾く	敵♥-1。敵の"防ぐ"無効。その時敵♥-1。
抓る	敵♥-1。敵の"躱す"無効。その時敵♥-1。
撚る	敵♥-1。敵の"誘う"無効。その時敵♥-1。
突く	敵♥-1。自の["棒"-1]なら敵♥-2

探す	30%:爆弾 30%:強薬 30%:治薬 10%:全部（爆弾:"使う"と敵♥-10。強薬。治薬:"使う"と自♥+7）
持つ	30%:棒 30%:盾 （盾:"使う"と盾+5）
拾う	30%:砂 30%:石 30%:棒 10%:全部（砂:"使う"と[即]敵命中率が次回迄30%低下。石:[即]敵♥-1。棒:）
使う	ランダムで持っている物を使う

防ぐ	🛡+1。"盾"所有時🛡+1。成功後🛡+1。
躱す	50%の確率で次の相手の攻撃を回避する。成功時50%の確率で"拾う"発動。
休む	♥+3。次回"防ぐ","躱す","誘う","知る","笑う"不可。
誘う	[即]相手の使う札を自分が決める
知る	[即]相手の使う札を事前に知る　
嗤う	♥+1。敵が"誘う","知る"時に敵♥-2。敵が"探す","持つ","拾う","使う"時にそれを敵に返す。
挫く	相手の使う札を使用前に破棄する
```

```javascript
(function(){//HTML要素に_listeners/litten/unlistenを追加する。
function isEl(v) {return v instanceof HTMLElement}
function isStr(v) {return 'string'===typeof v || v instanceof String}
function isFn(v) {return 'function'===typeof v && !v.toString().match(/^class /)}
function isAFn(v) {return (v instanceof (async function () {}.constructor))}
function isHandler(v) {return isFn(v) || isAFn(v)}
function isBln(v){return 'boolean'===typeof v}
class Listener {
    static add(el, name, handler, useCapture=false) {// element, name, handler, options/useCapture
        if (!isEl(el)) {throw new TypeError('対象要素（HTMLElement）が必要です。')}
        if (!isStr(name)) {throw new TypeError('第一引数はイベント名（String）であるべきです。')}
        if (!isHandler(handler)) {throw new TypeError('第二引数はイベントハンドラ（Function/AsyncFunction）であるべきです。')}
        if (!isBln(useCapture)) {throw new TypeError('第三引数はuseCapture（Boolean）であるべきです。')}
        if (!('_listeners' in el)) {
            el._listeners = []
            el.removeAllEventListener = function() {
                this._listeners.map(l=>Listener.remove(this, ...l));
            }
        }
        const i = el._listeners.findIndex(l=>name===l.name && useCapture===l.useCapture);
        if (-1===i) {
            el._listeners.push({name:name, useCapture:useCapture, handlers:[handler]});
            el.addEventListener(name, handler, useCapture);
        } else {
            if (el._listeners[i].handlers.includes(handler)) {return} // 重複登録防止
            el._listeners[i].handlers.push(handler);
            el.addEventListener(name, handler, useCapture);
        }
    }
    // 条件に一致したリスナーを削除する
    static remove(el, name, handler, useCapture) {// elは必須。name/handler/useCaptureは任意(null/undefinedなら全件対象)
        if (!('_listeners' in el)){return}
        const targets = el._listeners.filter(l=>(name ? name===l.name : true) && ('boolean'===typeof useCapture ? useCapture===l.useCapture : true) && (handler ? l.handlers.includes(handler) : true));
        // ハンドラ[一|全]件削除
        const REMOVE = handler
            ? ((t)=>{
                console.log(t)
                const i = t.handlers.indexOf(handler);
                el.removeEventListener(t.name, t.handlers[i], t.useCapture);
                t.handlers.splice(i, 1);
            })
            : ((t)=>{
                t.handlers.map(l=>el.removeEventListener(t.name, l, t.useCapture));
                t.handlers.length = 0;
            });
        targets.map(t=>REMOVE(t));
        // リスナ削除(ハンドラ0件のリスナ削除)
        el._listeners = el._listeners.filter(l=>0!==l.handlers.length);
    }
}
// EventTarget <-- Node <-- Element  ElementはEventTargetからプロパティを継承している。
EventTarget.prototype.listen = function(name, handler, useCapture=false) {//addEventListenerのハンドラ管理版

};
EventTarget.prototype.unlisten = function(name, handler, useCapture) {//removeEventListenerのハンドラ管理版
    Listener.remove(this, name, handler, useCapture);
};
EventTarget.prototype.tell = function(event) {return this.dispatchEvent(event);}//dispatchEventの糖衣構文版

# 解答

````markdown
提示された構成とやりたいことを踏まえ、**「単一責任原則」に基づいた拡張性の高いディレクトリ構成とクラス設計**を提案します。

現在の `decision.js` は「戦闘シーン全体」を司る役割ですが、中身が複雑になるため、**「状態管理（Logic）」**と**「表示（UI）」**、そして**「カードの効果（Action）」**を分離するのが良いでしょう。

### 2. 各クラスの役割と設計指針

#### ① Battler (model/battler.js)
HeroとEnemyの共通データ構造です。
- `hp`, `shield`, `items` (爆弾、棒、薬、砂、石) を持つ。
- `statusEffects`: 「次回〇〇不可」などの状態異常を管理する配列。

#### ② Card (model/card.js)
カードは単なるデータ（名前、説明、効果タイプ）として定義します。
```javascript
const CARDS = {
    TATAKU: { name: '叩く', description: '敵♥-1。前回攻撃済みなら敵♥-1。', type: 'attack' },
    FUSEGU: { name: '防ぐ', description: '🛡+1。"盾"所有時🛡+1。', type: 'defense' },
    // ...
};
```

#### ③ ActionResolver (engine/action-resolver.js)
**ここが一番重要です。** 「どのカードが使われたか」を受け取り、Battlerの状態を書き換える「審判」の役割をさせます。
- `resolve(card, attacker, defender)` メソッドを持つ。
- 「前回攻撃したか」「盾を持っているか」などの条件判定をここで行います。

#### ④ BattleScene (battle-scene.js)
全体の進行（ターン管理）を行います。
1. **開始時**: 山札をシャッフル。
2. **ターン開始**: 山札から3枚引く。
3. **Heroの選択待ち**: UI（HandView）にボタンを表示。
4. **計算**: ActionResolverでHeroの行動を反映。
5. **Enemyの選択**: Enemyクラスにカードを選ばせ、計算。
6. **判定**: どちらかのHPが0なら `onNext()` (Finished) を呼ぶ。

---

### 3. 実装のヒント：カード効果の抽象化

カードの効果が「♥-1」や「次回不可」など多岐にわたるため、ActionResolverでは以下のようなパターンで実装すると綺麗です。

```javascript
class ActionResolver {
    static execute(card, actor, target, history) {
        switch(card.id) {
            case 'TATAKU':
                target.hp -= 1;
                if (history.lastActionType === 'attack') target.hp -= 1;
                break;
            case 'FUSEGU':
                actor.shield += 1;
                if (actor.items.hasShield) actor.shield += 1;
                break;
            // ...
        }
    }
}
```

### 4. 次のステップとしてのアドバイス

まずは、**「最小限の戦闘サイクル」**を動かすために以下の3つを優先して作ると良いです。

1.  **`Battler` クラス**: HPを減らすメソッドなどを持つ。
2.  **`BattleScene` の `#make`**: 
    - `Dom.tags` を使って、HeroとEnemyのHPを表示するエリアを作る。
    - ランダムに3つのボタン（手札）を生成する。
3.  **簡易的なターン処理**:
    - ボタンを押す -> 敵のHPが減る -> 敵がランダムに攻撃してくる -> 自分のHPが減る -> 表示更新。

これらが動いたら、「盾」や「特殊なカード効果」を `ActionResolver` に追加していくのがスムーズです。

### 4. 山札と手札の管理 (`Deck`)

「3枚引いて1枚選ぶ」「残りは捨てる」「足りなくなったらシャッフル」というルールをカプセル化します。

```javascript
class Deck {
    constructor(cards) {
        this.drawPile = [...cards]; // 山札
        this.discardPile = [];      // 捨札
        this.hand = [];             // 手札
    }

    draw(count) {
        if (this.drawPile.length < count) {
            this.reshuffle();
        }
        this.hand = this.drawPile.splice(0, count);
        return this.hand;
    }

    discardHand() {
        this.discardPile.push(...this.hand);
        this.hand = [];
    }

    reshuffle() {
        this.drawPile.push(...this.discardPile);
        this.discardPile = [];
        // ここでシャッフルアルゴリズムを実行
        this.drawPile.sort(() => Math.random() - 0.5);
    }
}
```

### 1. `applyEffect` の返却値は何のため？

> `return { damage, heal };` が気になる。

これは**「UI（演出）への通知」**のためです。
`context` 内の `Battler` の数値を書き換えるのは「内部状態の更新」ですが、それだけだと画面上では「いつの間にかHPが減っていた」という状態になります。

*   **内部処理**: `enemy.life -= 3`
*   **返却値**: 「3ダメージ与えたよ！」という情報をUIに渡す。
*   **UI処理**: 敵の画像の上に「-3」という数字をポップアップさせたり、ログに「Heroの攻撃！敵に3ダメージ！」と表示したりする。

つまり、**「何が起きたか」を人間（プレイヤー）に伝えるためのデータ**です。

### 2. テキストと効果の二重管理を防ぐには？

> 効果とテキストを二重管理したらミスりそうで嫌。

これを解決するには、カードに**「計算用のパラメータ」**を持たせ、説明文はそのパラメータを参照するようにします。

```javascript
class AttackCard extends Card {
    constructor() {
        super('ATTACK', '叩く');
        this.power = 1; // 効果量をプロパティとして持つ
    }

    // 説明文を動的に生成する
    get description() {
        return `敵♥-${this.power}。`;
    }

    applyEffect(context) {
        context.target.receiveDamage(this.power);
        return { damage: this.power };
    }

// js/scene/battle/model/deck.js
class Deck {
    constructor(masterCards) {
        this.masterCards = masterCards;
        this.drawPile = [];
        this.discardPile = [...masterCards];
        this.hand = [];
        this._reshuffle();
    }

    _reshuffle() {
        this.drawPile = [...this.discardPile];
        this.discardPile = [];
        // Fisher-Yates
        for (let i = this.drawPile.length - 1; i > 0; i--) {
            const j = Math.floor(Math.random() * (i + 1));
            [this.drawPile[i], this.drawPile[j]] = [this.drawPile[j], this.drawPile[i]];
        }
    }

    draw(count) {
        const drawn = [];
        for (let i = 0; i < count; i++) {
            if (this.drawPile.length === 0) this._reshuffle();
            if (this.drawPile.length > 0) drawn.push(this.drawPile.pop());
        }
        this.hand = drawn;
        return drawn;
    }
}

### 1. `DOMException: Operation is not supported` の正体

このエラーは、`document.createElement(name, options)` の第2引数 `options` に、**`undefined` や空のオブジェクト、あるいは `is` プロパティが不正な値**で渡されたときに発生します。

特に、`is` プロパティ（Custom Elementsの拡張用）は、**値が存在しない場合は引数ごと渡さない**のが最も安全です。

#### `dom.tags.js` の修正案
以下のように、`is` が有効な文字列であるときだけ第2引数を渡すように書き換えてみてください。

```javascript
// dom.tags.js 内の tag 関数の中身
const tag = (ns, name, ...args) => {
    const [{is, ...props}, ...children] = 0<args.length && 'object'===typeof args[0] && '[object Object]'===Object.prototype.toString.call(args[0]) ? args : [{}, ...args];
    
    // 修正箇所：is が文字列の時だけ options を作成する
    const options = (typeof is === 'string' && is.length > 0) ? {is} : undefined;
    const dom = ns 
        ? (options ? document.createElementNS(ns, name, options) : document.createElementNS(ns, name))
        : (options ? document.createElement(name, options) : document.createElement(name));

    for (let [k, v] of Object.entries(props)) {
        if ('listeners'===k) {formatListeners(props[k]).forEach(l=>{dom.listen(...l)})}
        else {dom.setAttribute(k, v);}
    }
    dom.append(...children.map(c=>c));
    return dom;
}
```

### 1. `DOMException` の解決 (game.js)

**原因:** カスタム要素（Web Components）の `constructor` 内で `this.append()` を呼び出すことは仕様で禁止されています。このタイミングではまだ要素がドキュメントに接続されていないため、「サポートされていない操作」としてエラーになります。

**修正案:** `connectedCallback` を使い、要素が画面に表示される直前に子要素を追加するようにします。

```javascript
// js/game.js
class Game extends HTMLElement {
    constructor() {
        super();
        const names = 'Init Battle Finished'.split(' ');
        const shiftedNames = [...names];
        shiftedNames.push(shiftedNames.shift());
        const ons = shiftedNames.map(n=>()=>this.dispatchEvent(this._.events[n.toLowerCase()]));
        
        this._={
            events:names.map(n=>n.toLowerCase()).reduce((o,n)=>{o[n]=new CustomEvent(n); return o;},{}),
            scenes:names.reduce((o,n,i)=>{
                const cls = (new Function(`return ${n}SceneElement;`))();
                o[n.toLowerCase()]=new cls(ons[i]);
                return o;
            },{}),
        };

        // constructor での append は削除
        this.addEventListener('init', (e) => { /* ... */ });
        this.addEventListener('battle', (e) => { /* ... */ });
        this.addEventListener('finished', (e) => { /* ... */ });
    }

    // 要素がドキュメントに追加された時に呼ばれる
    connectedCallback() {
        if (!this.rendered) {
            this.append(...Object.values(this._.scenes));
            this.rendered = true;
            this.dispatchEvent(this._.events.init);
        }
    }
}
```

```javascript
class Game extends HTMLElement {
    constructor() {
        super();
        // 1. シーン名の定義（クラス名と一致させる）
        const names = 'Init Battle Finished'.split(' ');
        
        // 2. イベントの準備
        this._ = {
            events: names.reduce((o, n) => {
                o[n.toLowerCase()] = new CustomEvent(n.toLowerCase());
                return o;
            }, {}),
            scenes: {}
        };

        // 3. 次のシーンへ遷移する関数の準備
        // Init -> Battle, Battle -> Finished, Finished -> Init
        const shiftedNames = [...names];
        shiftedNames.push(shiftedNames.shift());
        const ons = shiftedNames.map(n => () => this.dispatchEvent(this._.events[n.toLowerCase()]));

        // 4. シーンインスタンスの生成
        names.forEach((n, i) => {
            try {
                const cls = (new Function(`return ${n}SceneElement;`))();
                this._.scenes[n.toLowerCase()] = new cls(ons[i]);
            } catch (e) {
                console.error(`クラス ${n}SceneElement が見つかりません。ファイル内で正しく定義されているか確認してください。`, e);
            }
        });

        // 5. イベントリスナーの登録
        this.addEventListener('init', () => {
            console.log('Scene: Init');
            this._hideAll();
            this._.scenes.init.show();
        });
        this.addEventListener('battle', () => {
            console.log('Scene: Battle');
            this._hideAll();
            this._.scenes.battle.show(); // ここが 'decision' のままだとエラーになります
        });
        this.addEventListener('finished', () => {
            console.log('Scene: Finished');
            this._hideAll();
            this._.scenes.finished.show();
        });
    }

    _hideAll() {
        Object.values(this._.scenes).forEach(s => s.hide());
    }

    connectedCallback() {
        if (!this.rendered) {
            // 子要素を追加
            this.append(...Object.values(this._.scenes));
            this.rendered = true;
            // 最初のシーンを表示
            this.dispatchEvent(this._.events.init);
        }
    }

### 1. なぜ FocusLooper が機能しなかったのか？

原因は主に2つ考えられます。

1.  **タイミングの問題**: `FocusLooper.setup()` を呼んだ時点では、まだ `Game` 要素（およびその中のボタン）が DOM に追加されていません。`MutationObserver` が動く前に `this.i = 0` を実行しても、対象が見つからないためフォーカスが当たりません。
2.  **Web Components との相性**: `FocusLooper` 内の `querySelectorAll` は、要素が追加された直後にはまだリストを更新できていない場合があります。

**解決策**: 
ゲームのような SPA（シングルページアプリケーション）では、**「シーンが切り替わった直後に、そのシーン内のメインボタンに明示的に `.focus()` を送る」** のが最も確実でシンプルです。

### 1. FocusLooper の改修

非表示要素（およびその子孫）を除外し、Tabキーを完全にトラップするように修正します。

```javascript
// js/util/focus-looper.js
(function(){
class FocusLooper {
    #FOCUSABLE_SELECTOR = [
        'a[href]', 'area[href]', 'input:not([disabled])', 'select:not([disabled])',
        'textarea:not([disabled])', 'button:not([disabled])', 'iframe', '[contenteditable]',
        '[tabindex]:not([tabindex^="-"])'
    ].join(',');

    constructor() {
        this._els = [];
        this._i = 0;
        
        // DOMの変化を監視してリストを更新
        const observer = new MutationObserver(() => this.refresh());
        window.addEventListener('DOMContentLoaded', () => {
            observer.observe(document.body, {
                childList: true, subtree: true, attributes: true,
                attributeFilter: ['class', 'style', 'disabled', 'hidden']
            });
            this.refresh();
        });
    }

    // 有効な（表示されている）要素だけを抽出
    refresh() {
        const all = Array.from(document.querySelectorAll(this.#FOCUSABLE_SELECTOR));
        this._els = all.filter(el => {
            // 1. offsetParent が null でない（物理的に表示されている）
            // 2. 自分自身または親要素に .is-hidden がついていない
            return el.offsetParent !== null && !el.closest('.is-hidden');
        });
    }

    setup() {
        window.addEventListener('keydown', (e) => {
            if (e.key === 'Tab') {
                e.preventDefault(); // ブラウザ標準のTab移動を完全に殺す
                this.move(e.shiftKey ? -1 : 1);
            }
        });
    }

    setFirst() {
        this.refresh();
        this._i = 0;
        if (this._els[0]) this._els[0].focus();
    }

    move(dir) {
        if (this._els.length === 0) return;
        this._i = (this._i + dir + this._els.length) % this._els.length;
        this._els[this._i].focus();
    }
}
window.FocusLooper = new FocusLooper();
})();
```
