````markdown
### 1. SceneManager システム要件定義（網羅）

これまでの対話から抽出した、実装すべき全要件を整理しました。

#### A. 識別子（ID）の制約
1.  **書式**: 小文字英数字と、単語区切りのハイフンのみを許容する。
    *   正規表現: `/^[a-z0-9]+(-[a-z0-9]+)*$/`
    *   例: `init-scene` (OK), `battle1` (OK), `-test` (NG), `Test` (NG), `a--b` (NG)
2.  **一意性（Scene）**: `SceneManager` に登録される場面IDは、プロジェクト全体で一意でなければならない。
3.  **一意性（Item）**: 一つの場面（Scene）の中に登録される部品IDは、その場面内で一意でなければならない。

#### B. ライフサイクルと状態管理
1.  **add(config)**:
    *   `build()` 実行前のみ呼び出し可能。実行後は `SceneError` を投げる。
    *   重複した場面IDの登録は `SceneError` を投げる。
2.  **build()**:
    *   登録された設定に基づき、`Scene` および `SceneItem` インスタンスを生成する。
    *   生成した `Scene` の要素（el）を `document.body` に接続する。
    *   `document.body` に `scene-container` クラスを付与する。
    *   一度しか実行できない。二度目は `SceneError` を投げる。
    *   登録された場面がゼロの場合は `SceneError` を投げる。
3.  **transition(id)**:
    *   `build()` 実行後のみ呼び出し可能。
    *   対象の場面を表示し、それ以外の全ての場面を非表示にする。
    *   表示直後に、その場面の `setup()` コールバックを実行する。
4.  **render(...configs)**:
    *   `build()` 実行後のみ呼び出し可能。
    *   指定された場面、およびその中の特定の部品を更新する。
5.  **clear()**:
    *   全ての場面要素を DOM から削除する。
    *   内部で保持しているインスタンス、設定、構築フラグをすべてリセットする。
    *   `document.body` から `scene-container` クラスを削除する。

#### C. DOM 属性とパフォーマンス
1.  **属性付与**:
    *   場面要素には `data-scene-id="場面ID"` を付与する。
    *   部品要素には `data-scene-parts-id="部品ID"` を付与する。
2.  **表示制御**:
    *   非表示状態の場面には `is-hidden` クラスを付与する。
    *   非表示状態の場面には `style.contentVisibility = 'hidden'` を適用する。
    *   表示状態の場面には `style.contentVisibility = 'auto'` を適用する。
3.  **更新負荷軽減**:
    *   `render` 実行中、対象場面の `contentVisibility` を一時的に `hidden` にし、完了後に `auto` に戻す。

#### D. コールバックの責務
1.  **build(target)**:
    *   `SceneItem`: インスタンス生成時に一度だけ実行。子要素の生成などを行う。
    *   `Scene`: `SceneManager.build()` 時に一度だけ実行。場面内の構造構築を行う。
2.  **setup(scene)**: 場面遷移（transition）のたびに実行。フォーカス制御などを行う。
3.  **render(el/scene, ...args)**: 描画更新時に実行。

#### E. 型バリデーション（Typer.ja.js の利用）
1.  全ての公開メソッドの引数、および `config` オブジェクトのプロパティに対して、実行時に型チェックを行う。
````

# 解答

````markdown
ご指摘いただいた「`el` の型を `Node` に広げる」点、および「`build()` 時の `DocumentFragment` による最適化」について、論理的な妥当性を確認しました。

### 1. 技術的考察

#### `el` の型について
`HTMLElement` ではなく `Node`（実質的には `Element`）を要求の基準とすることで、SVG やその他の名前空間を持つ要素も許容され、ライブラリの汎用性が高まります。
ただし、`Scene` クラス内で `setAttribute` や `style` プロパティを操作するため、これらを保持しない `TextNode` などが渡された場合には実行時エラーになります。利用者が `Dom.tags` を使う前提であれば、基本的には `Element` 以上の型が渡されるため、バリデーションの閾値としては `Node` で適切です。

#### `DocumentFragment` による最適化
`build()` において `DocumentFragment` を使用するのは、非常に有効なパフォーマンス改善策です。
*   **理由**: `document.body`（ライブな DOM）に対して繰り返し `appendChild` を行うと、その都度リフロー（再計算）が発生する可能性があります。`DocumentFragment` に全場面を一度蓄えてから一括で `body` に追加することで、ブラウザの描画負荷を最小限（1回のリフロー）に抑えることができます。
*   **`contentVisibility` との併用**: `DocumentFragment` は「初期接続」の負荷を下げ、`contentVisibility` は「接続後」の非表示場面のレンダリング負荷を下げます。両者を併用するのが最強の構成です。
````

````markdown
        const actType = this.type.get(value);
        if (actType !== type) {
            this.#throwError(type, value, name);
        }
        return true;
    }

    /**
     * 型の継承関係（instanceof）を含めて検証
     */
    static of(type, value, name) {
        this.type.valid(type);
        
        // 1. 完全一致なら合格
        const actType = this.type.get(value);
        if (actType === type) return true;

        // 2. 継承関係のチェック
        // プリミティブ型（String等）は get() で一致しなかった時点で不合格
        const isConstructor = (typeof type === 'function');
        const isPrimitiveCtor = [Boolean, Number, BigInt, String, Symbol].includes(type);
        
        if (isConstructor && !isPrimitiveCtor) {
            if (value instanceof type) return true;
        }

        this.#throwError(type, value, name);
    }

    static #throwError(type, value, name) {
        const typName = this.type.getName(type);
        const valName = this.value.getName(value);
        throw new TypeError(`${('string'===typeof name) ? `"${name}" の` : ''}型が不正です。期待: ${typName}, 実際: ${valName}。`);
    }
}
```
````

````markdown
### 2. `Typer.ja.js` の再設計（`of` メソッドの簡略化）

#### なぜプリミティブを `instanceof` から除外しようとしたか（JSの仕様）
JavaScript の仕様上、**プリミティブのリテラルは `instanceof` で `false` を返す**ためです。
*   `"abc" instanceof String` -> `false`
*   `123 instanceof Number` -> `false`

しかし、`Typer.of` の目的が「厳密一致 または 継承」であるならば、先に `TypeValue.get(value) === type` でリテラルの一致を確認し、その後に `instanceof` を実行すれば、プリミティブかどうかを個別に判定する必要はありません。

#### 修正版 `Typer.of`
ご提案通り、ロジックを統合してシンプルにします。

```javascript
    /**
     * 型の継承関係（instanceof）を含めて検証
     */
    static of(type, value, name) {
        this.type.valid(type);
        
        // 1. 厳密一致チェック (リテラル、null、undefined、NaN をカバー)
        if (this.type.get(value) === type) return true;

        // 2. 継承関係チェック (クラスの継承をカバー)
        // type がコンストラクタ関数であり、value がそのインスタンスであれば合格
        if (typeof type === 'function' && value instanceof type) return true;

        this.#throwError(type, value, name);
    }
```
これならば、`Typer.of(String, "abc")` はステップ1で合格し、`Typer.of(Element, span)` はステップ2で合格します。非常にDRYです。

### 3. `src/js/scene.js` の再設計（バリデーションの委譲と共通化）

ご指摘の通り、`validateId` を共通化し、`SceneManager` の重すぎるバリデーションを `Scene` や `SceneItem` に適切に分散（委譲）させます。

#### 修正のポイント
1.  **`validateId` の共通化**: スコープ内の関数として定義し、Scene/Item 両方で使用。
2.  **バリデーションの委譲**: `SceneManager` は `config` の大枠だけをチェックし、中身の詳細は `Scene` のコンストラクタが、さらにその中の部品は `SceneItem` がチェックする。
3.  **`Element` 型の採用**: `Node` ではなく `Element` で判定。

```javascript
import { Typer } from '../lib/typer/dist/esm/ja/bundle.js';

class SceneError extends Error {
    constructor(message, options) {
        super(message, options);
        this.name = "SceneError";
    }
}

// スコープ内共通IDバリデーション
const ID_REGEX = /^[a-z0-9]+(-[a-z0-9]+)*$/;
const validateId = (id, label) => {
    Typer.is(String, id, label);
    if (!ID_REGEX.test(id)) {
        throw new SceneError(`ID "${id}" の形式が不正です。小文字英数字と単語区切りのハイフンのみを使用してください。`);
    }
};

/**
 * 場面を構成する個別の部品
 */
class SceneItem {
    #id; #el; #build; #render;
    constructor(conf) {
        Typer.is(Object, conf, 'SceneItem config');
        validateId(conf.id, 'items[].id');
        Typer.of(Element, conf.el, 'items[].el'); // Element 以上の型を許容
        if (conf.build !== undefined) Typer.is(Function, conf.build, 'items[].build');
        if (conf.render !== undefined) Typer.is(Function, conf.render, 'items[].render');

        this.#id = conf.id;
        this.#el = conf.el;
        this.#build = conf.build;
        this.#render = conf.render;

        this.#el.setAttribute('data-scene-parts-id', this.#id);
        if (this.#build) this.#build(this.#el);
    }
    get id() { return this.#id; }
    get el() { return this.#el; }
    render(...args) {
        if (this.#render) this.#render(this.#el, ...args);
    }
}

/**
 * 場面（画面全体）の定義
 */
class Scene {
    #id; #el; #build; #render; #setup; #items = new Map();
    constructor(conf) {
        Typer.is(Object, conf, 'Scene config');
        validateId(conf.id, 'id');
        this.#id = conf.id;
        this.#el = conf.el || document.createElement('div');
        Typer.of(Element, this.#el, 'el');

        this.#build = conf.build;
        this.#render = conf.render;
        this.#setup = conf.setup;

        if (conf.items !== undefined) {
            Typer.is(Array, conf.items, 'items');
            this.#registerItems(conf.items);
        }

        this.#el.setAttribute('data-scene-id', this.#id);
        this.#el.classList.add('is-hidden');
    }

    get id() { return this.#id; }
    get el() { return this.#el; }
    get items() { return this.#items; }

    #registerItems(items) {
        items.forEach((i, idx) => {
            // SceneItem のコンストラクタにバリデーションを委譲
            if (this.#items.has(i.id)) {
                throw new SceneError(`場面 "${this.#id}" 内で部品ID "${i.id}" が重複しています。`);
            }
            this.#items.set(i.id, new SceneItem(i));
        });
    }

    build() { if (this.#build) this.#build(this); }
    setup() { if (this.#setup) this.#setup(this); }
    
    render(data = {}) {
        // 描画更新時の contentVisibility 制御
        this.#el.style.contentVisibility = 'hidden';

        if (data.items) {
            data.items.forEach(({id, args}) => {
                const item = this.#items.get(id);
                if (!item) throw new SceneError(`場面 "${this.#id}" に部品ID "${id}" は存在しません。`);
                item.render(...args);
            });
        }
        if (this.#render) this.#render(this, data.args || []);

        this.#el.style.contentVisibility = 'auto';
    }
}

/**
 * 場面全体の管理者
 */
export class SceneManager {
    static #configs = new Map();
    static #instances = new Map();
    static #isBuilt = false;

    static add(config) {
        if (this.#isBuilt) throw new SceneError("構築(build)後は場面を追加できません。");
        Typer.is(Object, config, 'config');
        validateId(config.id, 'id');
        
        if (this.#configs.has(config.id)) {
            throw new SceneError(`場面ID "${config.id}" は既に登録されています。`);
        }
        // 詳細なバリデーションは build() 時のインスタンス生成プロセスに委譲
        this.#configs.set(config.id, config);
    }

    static build() {
        if (this.#isBuilt) throw new SceneError("既に構築済みです。");
        if (this.#configs.size === 0) throw new SceneError("登録された場面がありません。");

        document.body.classList.add('scene-container');
        const fragment = document.createDocumentFragment();
        
        for (const conf of this.#configs.values()) {
            const scene = new Scene(conf); // ここで内部のバリデーションが走る
            scene.build();
            fragment.appendChild(scene.el);
            this.#instances.set(scene.id, scene);
        }
        
        document.body.appendChild(fragment);
        this.#isBuilt = true;
    }

    static transition(id) {
        this.#ensureBuilt();
        const target = this.#instances.get(id);
        if (!target) throw new SceneError(`場面ID "${id}" は存在しません。`);

        for (const s of this.#instances.values()) {
            s.el.classList.add('is-hidden');
        }
        target.el.classList.remove('is-hidden');
        target.setup();
    }

    static render(...configs) {
        this.#ensureBuilt();
        configs.forEach(conf => {
            Typer.is(Object, conf, 'render config');
            const scene = this.#instances.get(conf.id);
            if (!scene) throw new SceneError(`場面ID "${conf.id}" は存在しません。`);
            scene.render(conf);
        });
    }

    static clear() {
        this.#instances.forEach(scene => {
            if (scene.el && scene.el.parentNode) scene.el.remove();
        });
        this.#configs.clear();
        this.#instances.clear();
        this.#isBuilt = false;
        document.body.classList.remove('scene-container');
    }

    static get(id) {
        this.#ensureBuilt();
        const s = this.#instances.get(id);
        if (!s) throw new SceneError(`場面ID "${id}" は存在しません。`);
        return s;
    }

    static #ensureBuilt() {
        if (!this.#isBuilt) throw new SceneError("場面が構築されていません。");
    }
}
````
