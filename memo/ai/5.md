0.md〜4.mdに以前までのセッションでのやり取りが記録されている。これを引き継げ。

最後の解答への返答を以下にする。

`let SceneManager;`を削除したらエラー無く実行され全件合格した。

頼む。

# 解答

````markdown
承知いたしました。`SceneManager.add` における **`config.el` の型チェック** に関する `describe` ブロックを提示します。

このテストでは、`el` プロパティが指定されている場合に、それが `HTMLElement` のインスタンスであるかどうかを `Typer.is` が正しく検証しているかを確認します。

### test/scene.js (config.el 型チェック)

```javascript
                describe("config.el の型チェック", () => {
                    test.each([
                        ["null", null, "Null"],
                        ["数値", 123, "Number"],
                        ["文字列", "string", "String"],
                        ["Boolean", true, "Boolean"],
                        ["純粋なオブジェクト", {}, "Object"],
                        ["配列", [], "Array"],
                        ["関数", () => {}, "Function"],
                    ])("el が %s の場合は TypeError を投げる", (_, v, act) => {
                        const config = { id: 'test-scene', el: v };
                        const expectedMessage = `"el" の型が不正です。期待: HTMLElement, 実際: ${act}。`;
                        
                        expect(() => SceneManager.add(config)).toThrow(TypeError);
                        try {
                            SceneManager.add(config);
                        } catch (e) {
                            expect(e.message).toBe(expectedMessage);
                        }
                    });
                });
```

### 注意事項
現在の `src/js/scene.js` の `SceneManager.add` 実装（4.md時点）では、`config.el` のバリデーションが記述されていません。そのため、このテストを実行すると**失敗（Fail）**することが予想されます。

テストをパスさせるためには、`src/js/scene.js` の `SceneManager.add` メソッド内に以下のバリデーションを追加する必要があります。

```javascript
// src/js/scene.js 内の SceneManager.add 修正イメージ
static add(config) {
    if (this.#isBuilt) throw new SceneError("...");
    Typer.is(Object, config, 'config');
    Typer.is(String, config.id, 'id');
    // 以下の行を追加
    if (config.el !== undefined) Typer.is(HTMLElement, config.el, 'el');
    
    // ...以下略
}
```

このテストブロックを追加して実行し、結果を教えていただけますでしょうか。次は **各コールバック（build, render, setup）の型チェック** に進む予定です。````
