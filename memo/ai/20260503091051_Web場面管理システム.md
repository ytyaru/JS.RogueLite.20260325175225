# Web場面管理システム

Webアプリは複数の異なる場面を入れ替えることがある。この場面を管理する仕組みについて構想を練る。

<!-- more -->

## 場面

場面は常にひとつしか表示されない。

場面とは物理的には画面全体である。但し一つの役割を担った画面である。アプリは複数の異なる場面を遷移することでアプリ全体の役割を果たす。

### 部品

場面はそれを構成する部品を持つ。

## 場面遷移

各場面には遷移する／遷移されるの関係がある。それは単純な順序かもしれないし、双方向かも知れないし、任意の指定した場面かもしれない。

`contentVisibility`など適切に表示しつつ、表示負荷を下げる努力をすべき。

```css
[data-scene-id] {
  position: absolute; /* 重ね合わせることで空白領域を作らない */
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
}
[data-scene-id].is-hidden {
  content-visibility: hidden; /* レンダリングをスキップして高速化 */
  contain-intrinsic-size: 100vh; 
  pointer-events: none; /* クリックや選択を無効化して下の層へ通す */
  user-select: none;    /* テキスト選択も無効化 */
}
```

## HTML構造

```html
<html>
  <body class="scene-container">
    <div data-scene-id="start">
      <button data-scene-parts-id="start"></button>
    <div>
    <div data-scene-id="battle">
      <button data-scene-parts-id="start"></button>
    <div>
  </body>
</html>
```

* 場面管理はbodyをrootとする
* rootには`class`属性値に`scene-container`を付与する
* root直下には各場面が配置される
* 場面は`data-scene-id`属性を持ち、一意の値を持つ
* 場面は`data-scene-parts-id`属性を持つ要素を持つ
* `data-scene-parts-id`属性値は、一意の値を持つ

### 一意保証

画面もその部品も必ず一意の識別子を必要とする。もし一意の名前でないなら`Scene.add()`の時点で例外発生する。`get()`,`transition()`,`render()`などのAPIは使えない。

IDを指定する全場面において、存在しない識別子を指定したらその時点で例外発生すること。これにて一意保証する。存在しないIDを無視するようなことはしない。意図せず無視されることは意図と違うから即座に例外発生して気づかせるべきである。

### 生存期間

全場面とその部品の生存期間は同一セッション内の間である。ブラウザの更新ボタンを押下されるまで同一インスタンスを使い回す。

* 全場面とその部品は`Scene.add()`内で`new`され内部保持する
* 一度`new`されたHTMLElementは`remove()`されず保持し続ける（`remove()`されたら例外発生する）
* ブラウザの更新ボタンやブラウザの再起動をしない限り同一インスタンスを使い回す

### 構成

画面部品は識別子によって取り出すことができる。それを用いて複雑な画面構造を構築することができる。

```
Scene.make({
  scene: '場面識別子',
  make:(s)=>{
    s.el.appendChild(Dom.tags.div({class:'item-container'}, s.items.get('場面識別子')));
  }
})
```

### 機能

* 画面要素の一意保証
* 指定画面・部品の存在確認
* 指定画面・部品の取得
* 指定画面へ遷移
* 指定画面の更新

## API

```javascript
// 画面とその部品を網羅する（一意確認、HTMLElement型確認）。buildで構築する。
Scene.add({
  scene: ['場面識別子', Dom.tags.div()], // '場面識別子'だけでも良い
  parts: [
    ['部品識別子', Dom.tags.button({
      class:'some',
      listeners:{click:()=>{Scene.transition('場面識別子')}}},
      '開始')],
  ],
  build:(s)=>{// 場面構築（要素の順序や配置や構造(CSS用のdivを入れる)を成す）
    s.el.appendChild(Dom.tags.div({class:'item-container'}, s.items.get('場面識別子')));
  },
  setup:(s)=>{// 場面遷移したら呼ばれる
    s.items.get('場面識別子').focus();
  }
}, {...}, ...);
Scene.has('場面識別子'); // boolean
Scene.get('場面識別子').el; // HTML要素
Scene.get('場面識別子').parts.has('部品識別子'); // boolean
Scene.get('場面識別子').parts.get('部品識別子'); // HTML要素
Scene.transition('場面識別子'); // 場面遷移
Scene.render({ // 場面更新
  scene: '場面識別子', 
  parts: [
    ['部品識別子', 'textContent/innerText/innerHTML値']
  ]},
  render:(s)=>{
    s.items.get('部品識別子').textContent = '';
    s.items.get('部品識別子').innerText = '';
    s.items.get('部品識別子').innerHTML = '';
    s.items.get('部品識別子').value = '';
    s.items.get('部品識別子').disabled = true;
  },
{...}, ...);
```

### 場面更新

場面更新は一度に複数の項目を同時に更新することで負荷を下げて応答速度を向上させる。そのために`contentVisibility`を使う。

```
const S = Scene.get('場面識別子');
const s = S.el;
s.style.contentVisibility = 'hidden';
S.parts.get('部品識別子').textContent = '更新値';
S.parts.get('部品識別子').textContent = '更新値';
s.style.contentVisibility = 'auto';
```

尚、DocumentFragmentは要素を再生成せねばならず負荷が高いため却下。単純なHTML要素だけならまだしも、`onclick`などのイベントまで再生成せねばならず、メモリリークの頻度が高まるため。

場面更新は、基本的に対象要素を取得して、それに対して`textContent`, `innerText`, `innerHTML`, `value`の値を変更することで実現する。どの要素の時にどの値を更新すべきかは、`tagName`で判断できるだろう。`input`なら`value`、`span`なら`textContent`など。だが、それらは状況次第で変わる可能性がある。たとえば`span`の内容を`innerHTML`で更新したいこともあるはずだ。`<b>太字<b>`のようなことをするために。だが、セキュリティの問題がある。故に`textContent`にしたい場合がある。

そこで、`parts:`により値を直接指定する場合は、なるだけセキュリティの高い`value`属性値や`textContent`のみとする。それ以外の操作をしたい場合は、`render(s)`を使うこととする。尚、実行順は`parts:`, `render(s)`の順とする。

```
Scene.render({ // 場面更新
  scene: '場面識別子', 
  parts: [
    ['部品識別子', 'textContent/innerText/innerHTML値']
  ]},
  render:(s)=>{
    s.items.get('部品識別子').textContent = '';
    s.items.get('部品識別子').innerText = '';
    s.items.get('部品識別子').innerHTML = '';
    s.items.get('部品識別子').value = '';
    s.items.get('部品識別子').disabled = true;
  },
{...}, ...);
```

更新処理の前後には`contentVisibility`を挟んで再描画の負荷を最小限にすること。

### 場面遷移

`Scene.transition('場面識別子');`により指定した場面に遷移する。これは`data-scene-id`を持つ要素に対してclass属性値`is-hidden`を付け外しすることで実現する。

* 場面は必ず1つ表示する状態のみである
* 処理の前後には`contentVisibility`を挟んで再描画の負荷を最小限にすること
