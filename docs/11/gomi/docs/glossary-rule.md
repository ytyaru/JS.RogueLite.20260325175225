# 用語・表記規則 (Glossary Rules)

本ドキュメントは、札の説明文（全文および要約）を構築するための論理的規則を定義する。

## 1. 文体と構造 (Sentence Structure)
- **主語の省略**: 札の使用者（自分）が主語である場合は記述を省略する。
- **自動詞の採用**: 意志や干渉を排し、数値の変動を「増す」「減る」という現象として記述する。
- **目的語の順序**: `[対象]の[要素]は[数値][操作]` の順を基本とする。
- **ターゲット継続**: 同一札内で対象が連続する場合、二つ目以降の対象名は省略する。

## 2. 二重増減の表記 (Double Delta Logic)
「値を操作する量」を操作する場合、以下の専門用語を用いて「を」の連続を回避する。
- **命減量**: 生命点が減る量（ダメージ量）。
- **命増量**: 生命点が増える量（回復量）。
- **体減量**: 身体点が減る量。
- **体増量**: 身体点が増える量。
- **記述例**: 「身体点の減量は1増す」 (身体点を減らす量を1増やす、の意)

## 3. 効果要約の物理制約 (Summary Constraints)
最小画面（幅240px / 3ボタン / 1ボタン80px）における制約。
- **基本単位**: 12pxフォント使用時、1行あたり全角6文字（半角12幅）を限界とする。
- **行数**: 最大2行までを許容する。
- **優先順位**: 
    1. 数値変動（例：`敵♥-1`）
    2. 特性ラベル（例：`[連斬]`）
- **改行ルール**: 1行（12幅）に収まらない場合、特性ラベルの直前で改行を行う。

## 4. 国際化と記号 (i18n & Symbols)
- **日本語**: 助詞を含めスペースなしで記述する。
- **英語**: 要素（主体・属性・数値）の区切りに半角スペースを1つ入れる。
- **短縮記号**: 
    - 主体: `S` (Self), `O` (Opponent)
    - 属性: `♥` (Life), `♡` (Body)
    - ※記号が使用不能な環境に限り `L` (Life), `B` (Body) を代用する。
- **演算子**: `+`, `-` を使用し、増減を直感的に示す。
