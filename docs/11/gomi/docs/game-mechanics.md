# ゲームメカニクス (Game Mechanics)

本ドキュメントでは、本ゲームにおける勝利条件、リソース、および札の相互作用による論理的な面白さの根拠を定義する。

## 1. 基本ルール
- **対戦形式**: 1対1のターン制バトル。
- **勝利条件**: 相手の `life` を 0 以下にする。
- **初期状態**: 自他共に `life: 10 / maxLife: 10`。

## 2. リソース管理
- **life (現在ライフ)**: 0になると敗北する数値。
- **maxLife (最大ライフ)**: `life` を回復できる上限値。
- **札の配布**: 
    - 全4種類の札から、毎ターンランダムに3枚が手札として配布される。
    - 特定の1枚が手札に含まれる確率は 75% である。

## 3. 札の仕様
各札は以下の論理に基づいて機能する。

### 斬る (SLASH)
- **基本効果**: 敵の `life` を 1 減らし、敵の `maxLife` を $N$ 減らす。
- **累積ロジック**: 
    - この札を連続で選択している間、$N$ は 1 ずつ増加する（1 → 2 → 3）。
    - $N$ の上限は 3 とし、4連続以降も 3 を維持する。
    - 他の札（捨て身、薬）を選択した瞬間、$N$ は 1 にリセットされる。
- **制約**: `maxLife` の下限値は 1 とする。

### 捨て身 (SACRIFICE)
- **基本効果**: 敵の `life` を 3 減らし、自分の `life` を 1 減らす。
- **副作用**: 「斬る」の累積をリセットする。

### 薬 (MEDICINE)
- **基本効果**: 自分の `life` を 3 増やす。
- **制約**: `maxLife` を超えて回復することはできない。
- **副作用**: 「斬る」の累積をリセットする。

### 自然治癒 (AUTO_HEAL)
- **基本効果**: [即時発動] 自分の `life` を 1 増やす。
- **特性**: 
    - 手札に配布された時点で自動的に発動し、プレイヤーの選択を介さない。
    - プレイヤーの選択ではないため、「斬る」の累積をリセットしない。

## 4. 戦略的葛藤の構造
本ゲームの意思決定は、以下の3つの価値判断の対立によって成立する。

1.  **投資 (斬る)**: 
    - 短期的にはダメージ効率が悪いが、敵の `maxLife` を削ることで「未来の回復」を無効化し、詰みの状態へ追い込む。
2.  **即物 (捨て身)**: 
    - 自分の `life` を犠牲に、最短で勝利を目指す。敵の `maxLife` が削りきられる前の短期決戦に有効。
3.  **維持 (薬 / 自然治癒)**: 
    - 敗北を回避するための遅延行為。特に「自然治癒」は「斬る」の投資を継続しながら回復できる唯一の幸運要素となる。
