# プロジェクト方針

## 目的

ゲームを作り上げること。

## 方針

* プロジェクト完遂を最優先する
* 実装量を減らす
* ゲームが成立する最低要件だけは守る

## 経緯

私はゲームを作りたかった。しかしその計画は幾度となく破綻した。

1. 様々なゲーム要素を持った複雑で面白いゲーム
2. レスポンシブやタッチ操作、多HID対応など誰でも利用できるユーザビリティを備えたアプリ
3. トランスパイルとポリフィルによる互換性を持ったアプリ

現代において、およそ必要とされる要素の実装に尽く失敗した。そして理解した。私には世間一般で通じるレベルのゲームは作れないと。

頓挫する度に目標を大幅に下方修正することを繰り返してきた。そして今、ついにここまで堕ちた。

## ゲームの定義

じゃんけんやダイス投げよりは戦略性があるもの。これをゲームとする。以下の条件を満たす最小構成をゲームとする。

* 完全ランダムではない
* 完全論理確定ではない

### ゲームメカニクス

戦略性が存在することを論理的に説明する。

どう考えてもじゃんけんに毛が生えた程度だが、それでも完全ランダムではなければ、それでいい。

## ユーザビリティ

スマホ対応を切り捨てる。現代においてそれはもう成立しない。しかし私にはそれを実装する能力はない。よってスマホ対応を切り捨てる。

1. スマホ対応
2. レスポンシブ対応
3. 多HID対応（ゲームパッド、ポインティングデバイス（ペンタブ、タッチ））

古きゆかしい以下のみ対応する。

* PC
* マウス
* キーボード

### スマホ対応の難しさ

スマホにはフォーカスがない。全てをジェスチャーで実装せねばならない。もしマウスならばホバーによって詳細説明の表示ができたし、クリックによってその意思決定ができた。だがスマホは、タップによって詳細説明の表示をし、スワイプによってその意思決定をせねばならない。

マウス操作は特定のイベントで簡単に実装できる。だが、スマホのジェスチャーは、同一イベント内においてタイマーなどを駆使して自力で実装せねばならない。残念ながら、今はまだそこまで対応する余裕がない。

また、イベント実装だけでなく、それを伝えるアニメーション表示も必要だ。スワイプできるUI要素であることを伝えるために動く矢印を表示するなどだ。こちらのほうが実装難易度が高いだろう。同じく実装する余裕がない。

更に、上記の詳細説明やアニメは画面サイズに応じて最適化する必要がある。レスポンシブ対応まで考慮せねばならない。これらをすべて同時に叶える必要があり相互に絡み合うため複雑化し、その実装難度は跳ね上がる。私には無理だ。

## 互換性

一切考慮しない。私の開発環境はFirefox142だ。本来ならばトランスパイルやポリフィルでES5の古いブラウザや、他のブラウザ種別でも動作するようにしたかった。だが、そうした互換性を一切考慮しない。

## 課題

上記を前提として、画面設計や国際化は未だに夢見ている。きっとできるはずだと。しかしこれまで破綻を繰り返してきたパターンに酷似していることから、また破綻することが予想される。それでも挑戦するのが本プロジェクトの課題である。

おそらく本プロジェクトにおいて画面設計や国際化の詳細化をする。そして破綻し、何らかの妥協点を見出して、再び今より劣化したプロジェクトを新たに立ち上げることになるだろう。
```
