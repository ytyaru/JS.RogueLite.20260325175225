# 画面設計書 (Screen Design)

## 1. 共通仕様
- **解像度**: 240 x 320 px (固定)
- **フォント**: 等幅フォント (Monospace)
- **基本サイズ**: 16px (漢字の視認性を担保する最小サイズ)
- **強調サイズ**: 28px (ライフ数値用)
- **補足サイズ**: 12px (ターン数用)
- **字間 (letter-spacing)**: 0.05em (詳細説明のみ適用)
- **行間 (line-height)**: 1.5em (詳細説明のみ適用。他は 1.0em)

## 2. 垂直配置の計算 (240x320)
- **Header**: 20px。名前(16px)とターン数(12px)を配置。
- **Status**: 40px。ライフ数値を28pxで表示。
- **Action**: 60px。札ボタン(16px)を配置。
- **Gap**: 4px。`#detail` の `margin-top` により確保。
- **Detail**: 1fr。残りの全領域。長文時はスクロールバーを表示。

## 3. 詳細説明の表記ルール
- **BudouXの採用**: `word-break: keep-all` と `overflow-wrap: anywhere` を併用し、文節単位の改行を強制する。
- **禁則処理**: `line-break: strict` を使用し、句読点が行頭に来ることを防ぐ。
- **約物の最適化**: `text-spacing-trim: trim-start;` を記述。対応ブラウザでの「追い込み（句読点の半角化による行末収容）」を期待して残置する。

## 4. 視覚的ヒエラルキーの論理
1. **自他識別**: 最上部の名前(16px)で左右の領域を定義。
2. **状況確認**: 28pxのライフ数値で現在の生命点と身体点を提示。
3. **意思決定**: 60px高のボタン(16px)で行動を選択。
4. **補助情報**: 最下部の説明文(16px)でルールの詳細を補完。
