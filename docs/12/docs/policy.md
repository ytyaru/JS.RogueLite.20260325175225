# プロジェクト憲章 (Project Policy)

## 1. 開発の動機 (Motivation)
- **創作の喜び**: 自分の思い通りに動く玩具を作る。

## 2. ユーザーへの約束 (User Policy)
- **アクセシビリティの最大化**: 
    - デバイス（PC/スマホ/タブレット）やOSを問わず、ブラウザさえあれば動作する。
    - レスポンシブ対応（画面サイズに応じた最適化）を行う。
    - 多様な入力デバイス（タッチ、マウス、ペンタブ、キーボード、ゲームパッド）に対応する。
    - オフライン動作（PWA）を可能とし、ネットワーク環境に左右されない。
- **純粋な遊戯性の追求 (Pure Game)**:
    - ゲーム体験を阻害する「ゲーム外のノイズ」を一切排除する。
    - 営利・宣伝・マーケティング活動および迷惑行為を一切行わない（販売、課金、広告、ステマ、個人情報の要求、登録促進、トラッキング、クリプトジャッキング、マルウェア、盗聴、盗撮、その他犯罪や嫌がらせ等）。
- **透明性と安全性による証明**:
    - 上記の「迷惑行為の排除」が嘘偽りなく実行されていることを証明する手段として、オープンソース（AGPL）を採用し、誰でもコードを検証可能にする。
- **自由の尊重**:
    - 特定のプラットフォームに依存せず、ユーザーが自由に管理・実行できる。
    - AGPLライセンスに基づき、ソースコードの閲覧・改変・再配布の自由を**法律で保証する**。これにより、特定の個人や企業による利用権の独占を防ぎ、永続的に誰でも無料で実行可能な状態を維持する。

## 3. 実装方針 (Implementation Policy)
- **優先順位の定義**: 
    1. 完遂（完成させること）
    2. 保守性（破綻させないこと）
    3. 互換性（広く使えること）
    の順で優先する。
- **完遂の重視**: 
    - 頓挫を避けるため、最小構成（1プレイ10秒）の実装をゴールとし、確実に完成させる。
    - **アセットの最小化**: 開発リソース（能力・時間）の不足による頓挫を防ぐため、画像・音声を使用せず、HTML/CSS/文字のみで表現する。
- **保守性と互換性の両立**: 
    - 保守性と開発効率を最大化するため、最新のJS機能（Private `#` 等）を用いて実装する。
    - 同時に、ビルドプロセス（Babel等）によるトランスパイルを行うことで、**ES2015 (ES6)** 以降のブラウザで動作する互換性を確保する。
    - これにより、「開発時のコード品質（保守性）」と「実行時の環境不問（互換性）」を矛盾なく両立したコードを出力する。
- **外部依存の管理**: 
    - 外部ライブラリを使用する場合は、その**非minify版（人間が読める状態）のソースコード全文**をプロジェクト内に保持する。これにより、ライブラリの配布終了やブラウザ仕様変更時にも自力で修正・維持できる体制を整え、長期的な動作安定性を確保する。
