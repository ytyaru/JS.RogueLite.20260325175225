# プロジェクト憲章 (Project Policy)

## 1. 開発の動機 (Motivation)
- **創作の喜び**: 自分の思い通りに動く玩具を作る。
- **完遂の重視**: 頓挫を避けるため、最小構成（1プレイ10秒）の実装をゴールとし、確実に完成させる。

## 2. ユーザーへの約束 (User Policy)
- **アクセシビリティの最大化**: 
    - デバイス（PC/スマホ/タブレット）やOSを問わず、ブラウザさえあれば動作する。
    - レスポンシブ対応（画面サイズに応じた最適化）を行う。
    - 多様な入力デバイス（タッチ、マウス、ペンタブ、キーボード、ゲームパッド）に対応する。
    - オフライン動作（PWA）を可能とし、ネットワーク環境に左右されない。
- **迷惑行為の排除**: 
    - ゲーム外の迷惑行為を一切行わない（販売、課金、広告、個人情報の要求、登録促進、トラッキング、クリプトジャッキング、マルウェア、盗聴、盗撮、その他犯罪や嫌がらせ等）。
- **透明性と安全性**:
    - オープンソース（AGPL）とし、コードの透明性を確保する。
- **自由の尊重**:
    - 特定のプラットフォームに依存せず、ユーザーが自由に管理・実行できる。
    - AGPLライセンスに基づき、ソースコードの閲覧・改変・再配布の自由を保証する。これにより、特定の個人や企業による利用権の独占を防ぎ、永続的に誰でも無料で実行可能な状態を維持する。

## 3. 実装方針 (Implementation Policy)
- **保守性の追求**: 自分が後で困らないよう、変更に強く、論理的に美しいコードを目指す。
- **アセットの最小化**: 開発リソース（能力・時間）の不足による頓挫を防ぐため、画像・音声を使用せず、HTML/CSS/文字のみで表現する。
- **動作環境の定義**: 
    - モダンなJS機能（Class, Promise, async/await）を必須とする。
    - Private記述（`#`）等の比較的新しい機能については、開発効率を優先して使用する。互換性については、必要に応じてビルドプロセス（Bun等）での対処を検討する。
- **依存の最小化**: 外部ライブラリの使用は、自前実装が困難な機能（複雑なジェスチャー制御等）に限定し、コアロジックの透明性を維持する。
